# v0.7.10 Spec — Cockpit Mini-Card: G2 校准 brier 速览

## 背景

v0.7.9 把 Platt + Isotonic 校准实验放到了 `/elo` 路由的 5-th tab (Calibrated)，
但 Cockpit (`#/` 路由) 没有显示。主人提"mini-card 加 brier，1h 不用切 tab 就能看到"。

## 现有痛点

1. `/api/elo/calibrated-predict/{home}/{away}` 每次都现场算 913 场 walkforward (~100ms)
2. Cockpit 主页面频繁刷新（每 60s scheduler）—— 拉 5-th tab 数据会卡
3. mini-card 不需要 home/away 维度，只需要"全局 G2 校准 brier 摘要"

## 目标

- 新增**轻量端点** `/api/elo/calibration-summary`
  - 不依赖 home/away
  - 用 **进程内 cache** (TTL 6h + 启动 lazy compute)
  - 返回 3 个 metric + 训练样本数 + 缓存时间戳
- Cockpit 加 1 个 mini-card
  - 标题 + 3 列 (Platt full / Platt walkforward / Isotonic walkforward)
  - 数字绿色 ↓ / 红色 ↑ (brier 越低越好)
  - 训练样本数 + cache 状态
  - 复用 `.cockpit-mini-card` 模板（Tailwind utility class）

## 范围

**In**:
- 1 新端点 `/api/elo/calibration-summary`
- 1 mini-card in Cockpit
- 1 unit test + 1 e2e
- spec + release
- commit + tag v0.7.10

**Out** (明确不做):
- 不写 prediction_log (mini-card 只读)
- 不暴露单场 calibrated 预测 (那是 /elo tab 范畴)
- 不加 admin 调权 (门槛 1.5pp 未过, 无市场)

## 端点设计

### Request
```
GET /api/elo/calibration-summary
```

### Response 200
```json
{
  "training_samples": 913,
  "platt_full_fit_pp": -0.69,
  "platt_walkforward_80_20_pp": -0.27,
  "isotonic_walkforward_80_20_pp": -0.23,
  "cache_ttl_seconds": 21600,
  "computed_at": "2026-06-17T02:17:00Z"
}
```

字段含义:
- `*_pp`: brier 改进百分点（负数 = 校准后 brier 变低 = 更好）
- `cache_ttl_seconds`: 缓存有效期 (6h)
- `computed_at`: ISO 8601 UTC, 客户端可显示"X 分钟前"

### Errors
- 500 Internal: walkforward 出错（数据缺失）

## 服务层设计

`app/services/calibration.py` 新增模块级 cache:
```python
_calibration_cache = {"data": None, "computed_at": None}
_CACHE_TTL_SECONDS = 21600  # 6h
```

公开函数:
```python
def get_calibration_summary(db: Session) -> dict:
    """带 6h 缓存的校准摘要。lifespan startup 预热一次。"""
    now = datetime.now(timezone.utc)
    if (_calibration_cache["data"] is not None
        and (now - _calibration_cache["computed_at"]).total_seconds() < _CACHE_TTL_SECONDS):
        return _calibration_cache["data"]
    # 否则现场算
    records = load_g2_records()
    platt_wf = walkforward_validate(records, 0.2)
    platt_full = evaluate_all(records, method="platt")
    iso_wf = isotonic_walkforward_validate(records, 0.2)
    data = {
        "training_samples": len(records),
        "platt_full_fit_pp": platt_full["brier_improvement_pp"],
        "platt_walkforward_80_20_pp": platt_wf["brier_improvement_pp"],
        "isotonic_walkforward_80_20_pp": iso_wf["brier_improvement_pp"],
    }
    _calibration_cache["data"] = data
    _calibration_cache["computed_at"] = now
    return data
```

注意: `evaluate_all` 和 `isotonic_walkforward_validate` 都需要先检查 v0.7.9 实际函数名。

## 风险

1. **缓存失效率** — 6h TTL, 但 walkforward 每次 100ms 现场算, 6h 内 N 次 Cockpit 刷新只算 1 次 → 风险低
2. **lifespan startup 调用顺序** — `get_calibration_summary` 需在 data/prediction_log_backfill.jsonl 准备好之后才有效 → 启动时若文件缺则 lazy compute 时会报错
3. **mini-card 颜色冲突** — Tailwind 已有 emerald/amber/cyan, 与 weight-sweep 不冲突
