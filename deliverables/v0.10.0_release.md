# v0.10.0 Release — Observability 观测能力落地

> 2026-06-17
> **直面主人 3 个真问题之 #1 部署/可用性**

## 主人 push back 背景

> 主人原话：
> 1. 没有公网访问入口（只本地 127.0.0.1:8000）
> 2. 没有 uptime 监控 / 告警
> 3. worldcup26.ir 挂了你怎么知道？（scheduler 是 best-effort）
> 4. 没有 CI/CD —— push 不会自动跑测试

**v0.10.0 解决 #2 + #3**（#1 公网部署靠 v0.12 + 主人买服务器, #4 CI/CD 靠 v0.12）。

## 不做什么（v0.10 边界）

- ❌ 公网部署（需主人买域名 + VPS + HTTPS，v0.12 Deployment Infra）
- ❌ Slack/邮件告警（v0.10 只暴露数据，告警靠主人外部脚本 polling /health）
- ❌ Prometheus metrics（口径不同，不做）
- ❌ CI/CD（v0.12 解决）
