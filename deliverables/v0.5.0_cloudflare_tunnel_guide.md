# v0.5.0 Cloudflare Tunnel 临时公网访问指南

> 版本：v0.5.0（临时演示）
> 日期：2026-06-15
> 状态：即用指南

---

## 一、适用场景

- 临时给其他人展示网站效果
- 不想购买服务器/域名
- 你的电脑能稳定开机运行本地服务

**限制**：
- URL 是临时的，每次重启 tunnel 会变
- 你电脑关机后网站无法访问
- 不满足“持续运行 + 定时同步”的长期需求

---

## 二、前置条件

1. 本地已运行 v0.4.0 服务：`http://127.0.0.1:8000`
2. 已能访问 Cloudflare（无需付费账号，免费即可）
3. Windows 系统（本指南以 Windows 为例）

---

## 三、操作步骤

### 步骤 1：下载 cloudflared

访问 Cloudflare 官方发布页：

```
https://github.com/cloudflare/cloudflared/releases
```

下载 Windows 64 位版本：`cloudflared-windows-amd64.exe`

下载后改名为 `cloudflared.exe`，放到一个固定目录，例如：

```
C:\tools\cloudflared.exe
```

### 步骤 2：把 cloudflared 加入环境变量（可选但推荐）

1. 右键“此电脑” → 属性 → 高级系统设置 → 环境变量
2. 在“系统变量”里找到 `Path`，点击编辑
3. 添加 `C:\tools`
4. 打开新的 PowerShell/CMD 窗口

### 步骤 3：启动本地服务

在项目根目录打开 PowerShell/CMD，运行：

```powershell
python scripts/start_server.py
```

确认本地可访问：

```powershell
curl http://127.0.0.1:8000/health
```

### 步骤 4：启动 Cloudflare Tunnel

在另一个 PowerShell/CMD 窗口运行：

```powershell
cloudflared tunnel --url http://127.0.0.1:8000
```

等待几秒，你会看到类似输出：

```
INF Connection registered connIndex=0 ip=198.41.192.227 location=SJC
INF Ingress proxy configuration:
INF + hostname: random-name.trycloudflare.com
INF |-- path: /
INF |-- service: http://127.0.0.1:8000
```

复制 `https://random-name.trycloudflare.com` 发给需要访问的人。

---

## 四、一键启动脚本

已准备 `scripts/start_tunnel.bat`，内容如下：

```batch
@echo off
echo [tunnel] 启动 Cloudflare Tunnel，映射到 http://127.0.0.1:8000
echo [tunnel] 请确保本地服务已启动：python scripts/start_server.py
cloudflared tunnel --url http://127.0.0.1:8000
pause
```

**用法**：

1. 把 `cloudflared.exe` 放到 `C:\tools\` 或项目 `scripts/` 目录
2. 先启动本地服务：`python scripts/start_server.py`
3. 双击运行 `scripts/start_tunnel.bat`

## 五、常见问题

### Q1：提示“cloudflared 不是内部或外部命令”

- 没加环境变量，或 cloudflared.exe 不在当前目录
- 解决：使用完整路径，如 `C:\tools\cloudflared.exe tunnel --url http://127.0.0.1:8000`

### Q2：tunnel 启动后 URL 打不开

- 检查本地 8000 端口是否真的启动了：`curl http://127.0.0.1:8000/health`
- 检查防火墙是否拦截
- 等待 5-10 秒再试

### Q3：URL 每次都不一样

- 这是临时 tunnel 的默认行为
- 如需固定域名，需要注册 Cloudflare 账号、添加域名、创建 named tunnel，步骤更复杂
- 固定域名方案见下一节

---

## 六、固定域名方案（进阶）

如果你已经有 Cloudflare 账号和域名，可以做长期稳定的 tunnel：

```powershell
# 1. 登录
cloudflared tunnel login

# 2. 创建 tunnel
cloudflared tunnel create wc2026

# 3. 配置 DNS（把 wc2026.yourdomain.com 指向 tunnel）
cloudflared tunnel route dns wc2026 wc2026.yourdomain.com

# 4. 创建配置文件 config.yml
# 内容见 Cloudflare 官方文档

# 5. 运行
cloudflared tunnel run wc2026
```

详细步骤参考：https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/

---

## 七、安全提醒

- 临时 tunnel 的 URL 是公开的，任何知道链接的人都能访问
- 不要把 Admin Token 泄露给不需要的人
- 演示结束后关闭 tunnel 窗口
