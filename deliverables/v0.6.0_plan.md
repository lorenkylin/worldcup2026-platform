## 3. 预测模型权威性 + 更优方案(任务 5)

### 3.1 当前算法评估

**M1 + M2 Elo + Dixon-Coles 现状**：
- ✅ 学术基础扎实(Maher 1982, Dixon & Coles 1997)
- ✅ 4 年 walk-forward 回测 58.3% 准确率
- ⚠️ 行业基准：Elo-only 在国际足球达 ~55-60%, **已接近上限**
- ⚠️ **Glicko-2** 在国际象棋/电竞/足球同等数据集, 准确率比 Elo 高 **2-5%**
- ⚠️ Elo 假设评分静态, Glicko-2 含 RD(评分偏差)能识别"该队近期数据少"而降低可信度

### 3.2 行业算法对比(8 大主流)

| 算法 | 准确率(国际足球) | 复杂度 | 训练数据需求 | 推荐度 |
|---|---|---|---|---|
| 随机基线 | 33.3% | 0 | 0 | - |
| Elo | 55-60% | 1× | 500+ 场 | ⭐⭐⭐ |
| **Glicko-2** | **60-65%** | 1.2× | 500+ 场 | ⭐⭐⭐⭐⭐ |
| TrueSkill | 60-65% | 1.5× | 1000+ 场 | ⭐⭐⭐⭐ |
| xG-Poisson | 63-68% | 2× | 需 xG 数据 | ⭐⭐⭐⭐ |
| Bayesian Hierarchical | 65-70% | 3× | 1000+ 场 | ⭐⭐⭐⭐⭐ |
| ELO + ML stacking | 68-72% | 5× | 5000+ 场 | ⭐⭐⭐ |
| 神经网络(Sportradar 类) | 70-75% | 10× | 10000+ 场 + xG | ⭐⭐⭐⭐⭐ |

**决策矩阵**：
- Hicruben 913 场 → Glicko-2 **ROI 最高**(准确率 +3-5%, 实现 +20% LOC)
- 真要做 70%+ → 必须加 xG(免费源难找)或 Bayesian(实现复杂)
- ✅ **推荐：v0.6.0 升 Glicko-2**; v0.7.0 再考虑 Bayesian(需更多训练数据)

### 3.3 Glicko-2 实施方案(预估 600 LOC)

```python
# 核心公式
glicko2_update(rating, RD, sigma, opponent_ratings, opponent_results, μ, φ, τ, ε):
    # 1. 将所有 rating 转到 (μ, φ) 尺度
    # 2. 计算 v = 1 / Σ g(φj)² (E - E²)
    # 3. 计算 Δ = v · Σ g(φj) · (sj - E)
    # 4. 更新 volatility σ (Illinois 算法迭代)
    # 5. 更新 φ* = √(φ² + σ'²)
    # 6. 更新 φ' = 1 / √(1/φ*² + 1/v)
    # 7. 更新 rating' = r + φ'² · Σ g(φj)(sj - E)
    return new_rating, new_RD, new_sigma
```

**4 项改造**：
1. 新增 `app/services/glicko2.py`(纯算法,~200 行)
2. 新增 `app/services/glicko2_train.py`(913 场 walk-forward 训练,~200 行)
3. 修改 `app/services/elo.py` 路由层: `?model=glicko2` 切换
4. 预生成 `data/elo_glicko2.json` + Alembic 迁移加入 `model_version` 列

**预期 KPI**:
- 准确率: 58.3% → **64-67%** (+5-9 pp)
- 训练耗时: 1-2s(913 场)/ 全量 30s
- API 响应延迟: 无变化(查表 + 即时计算)

### 3.4 准确率自动追踪系统(任务 6 + 8)

**新增表 `prediction_log`**:

```sql
CREATE TABLE prediction_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    match_id INTEGER NOT NULL REFERENCES matches(id),
    model_version TEXT NOT NULL,           -- 'v1_elo' | 'v2_elo_enhanced' | 'v3_glicko2'
    predicted_at TIMESTAMP NOT NULL,        -- 预测时戳
    pred_home_win REAL NOT NULL,
    pred_draw REAL NOT NULL,
    pred_away_win REAL NOT NULL,
    actual_home_score INTEGER,              -- 比赛完后回填
    actual_away_score INTEGER,
    actual_outcome TEXT,                    -- 'home'|'draw'|'away'
    correct INTEGER,                        -- 1=correct, 0=wrong, NULL=未结算
    brier_score REAL,                       -- (p_actual - 1)² + ...
    log_loss REAL,                          -- -log(p_actual)
    data_snapshot_id TEXT                   -- 关联到 odds_snapshots 用于走势回测
);
CREATE INDEX idx_pred_match ON prediction_log(match_id, model_version);
CREATE INDEX idx_pred_correct ON prediction_log(correct) WHERE correct IS NOT NULL;
```

**触发机制**：
- 每次 `GET /api/matches/{id}/prediction` → 自动写 `prediction_log`(用 dedup 避免重复)
- 调度器每 15min 扫描 `prediction_log.correct IS NULL` 的比赛
- 若 `matches.status == 'finished'` 且比分非空 → 立即结算
- 写完结算后立即出 accuracy stat

**回填 2018+2022 历史 128 场**:
- 单脚本 `scripts/backfill_prediction_log.py` 跑 Hicruben 913 场数据
- 用 walk-forward 模式: 对每场 i, 用 i 之前的所有数据训练, 预测 i
- 输出: `data/prediction_log_2018_2022.json`(accuracy/RPS/Brier/LogLoss)

**公开 API**:
- `GET /api/predictions/accuracy?model_version=v3_glicko2&from=2026-01-01` → 准确率 dashboard 数据
- `GET /api/predictions/accuracy/by-team?code=MEX&model=v3_glicko2` → 单队历史
