## 一、目标 & 收益

主人原话:
1. football-data.co API 接入(10 req/min + 15min 内存缓存,免 Key)
2. 赔率走势图表(Chart.js 单场赔率时间序列)
3. 球赛数据每 6 小时要定期刷新更新

| 目标 | 实现 | 收益 |
|---|---|---|
| football-data.co 接入 | 完整客户端 + 限速 + 缓存 + 异常分类 | 备用交叉验证源,弥补 wc26 偶发故障 |
| 赔率走势 | snapshot 表 + 6h 自动打点 + Chart.js 折线图 | 时间维度可视化,识别赔率漂移 |
| 6h 周期刷新 | 编排服务 + scheduler 注册 + lifespan startup | 无需人工干预,长跑后数据自动丰富 |

## 二、技术方案执行

### 决策记录(主人已确认)

| 决策点 | 选择 | 原因 |
|---|---|---|
| Q1:football-data.co 接入方式 | **完整接入** | 主人提供 free token,奴才写完整代码 |
| Q2:赔率快照数据驱动 | **6h 自动打点** | 零成本,自动有时间戳;未来接付费 API 激活波动 |
| Q3:6h 调度器范围 | **fb-data 元数据 + odds 快照** | scope 最小化,不动 wc26 主源 |

### 关键发现(奴才先 surface)

1. **football-data.co v4 实际需要 token**(免付费,但必须 header `X-Auth-Token`)—— 主人需在 .env 填
2. **football-data.co 免费层无赔率端点** —— 走势曲线初始平坦,需付费 API 才能激活真实波动
3. **SQLAlchemy JOIN 同名表歧义** —— 用 `aliased(Team)` 解决
4. **app.dependency_overrides 模块级污染** —— 改用 `autouse fixture + clear()` 隔离测试
5. **JS 三引号语法错误** —— 改用单行 `//` 注释避免 JS 解析失败

## 六、关键代码片段

### 6.1 football_data 客户端核心

```python
class FootballDataClient:
    BASE_URL = "https://api.football-data.org/v4"

    def _wait_for_rate_limit(self):
        """滑动窗口:Deque 记录 60s 时间戳,超过 10 req/min 时 sleep."""
        if len(self._request_times) < self.rate_limit_per_min:
            return
        now = time.monotonic()
        oldest = self._request_times[0]
        if now - oldest < 60:
            time.sleep(60 - (now - oldest) + 0.05)  # 50ms 缓冲
```

### 6.2 periodic refresh 幂等去重

```python
def take_odds_snapshots(db):
    now = datetime.now(timezone.utc)
    for odds in db.query(MatchOdds).all():
        # 同一 (match, bookmaker) 在 2s 窗口内已存在 → 跳过
        existing = db.query(OddsSnapshot).filter(
            OddsSnapshot.match_id == odds.match_id,
            OddsSnapshot.bookmaker == odds.bookmaker,
            OddsSnapshot.snapshot_at >= now - timedelta(seconds=2),
            OddsSnapshot.snapshot_at <= now + timedelta(seconds=2),
        ).first()
        if existing:
            skipped += 1
            continue
        db.add(OddsSnapshot(... source="periodic_6h"))
```

### 6.3 Chart.js 走势图核心

```javascript
async function loadOddsTrend(matchId) {
  const data = await api(`/matches/${matchId}/odds/history`);
  if (!data.has_history) {
    document.getElementById('odds-trend-empty').classList.remove('hidden');
    return;
  }
  // 收集所有时间戳(去重排序)
  const labels = [...new Set(
    Object.values(data.series).flatMap(s => s.map(p => p.t))
  )].sort();
  // 配色:每家公司一种颜色
  const palette = ['#3b82f6', '#ef4444', '#10b981', '#f59e0b', '#8b5cf6'];
  const datasets = data.bookmakers.map((bm, i) => ({
    label: bm, data: labels.map(t => data.series[bm].find(p => p.t === t)?.home_win),
    borderColor: palette[i % palette.length], spanGaps: true, tension: 0.25,
  }));
  _oddsTrendChart = new Chart(ctx, { type: 'line', data: { labels, datasets }, ... });
}
```

## 八、风险与已知限制

| 风险 | 影响 | 缓解 |
|---|---|---|
| football-data.co 缺赔率端点 | 走势曲线初始平坦 | 6h 自动打点 + 未来接付费 API |
| 6h 调度器与 15min 主同步冲突 | 可能短暂数据不一致 | refresh 只更新元数据,不创建/删除比赛(幂等) |
| odds_snapshots 表增长 | 长期可能膨胀 | 加了复合索引;后续可加 30 天 TTL 清理 |
| free token 需主人注册 | 没 token 时该模块 skip | .env 缺失时优雅降级,不影响其他功能 |
| Chart.js CDN 受限 | 沙箱环境可能拉不到 | jsdelivr 国内访问稳定;可换 unpkg |
