# v0.11.0 Spec — Forward-Testing

> 2026-06-17 — 距世界杯开赛 17 天
> **核心动机**: 主人 push back 所有 66.67% 都是历史回测, 不是 forward testing

## 1. 核心问题

| 现状 | 痛点 |
|---|---|
| 全部 66.67% 是 backtest 跑历史 | 不是 forward testing, 不知真实水平 |
| prediction_log 1860+ 行全是历史回填 | 跟开赛日预测结果无关 |
| 17 天内无 live 比赛 | 测不出真 forward |
| value_bet 算法从未在真赔率验证 | 收益不确定 |

## 2. 解决方案

### 2.1 prediction_log 区分 backfill vs live

加 2 字段:
- `is_live BOOLEAN DEFAULT 0` — True = 赛前实时预测, False = backfill
- `snapshot_group VARCHAR(40)` — 同一比赛同模型多次预测的快照组

### 2.2 写入策略

| 写入来源 | is_live | 说明 |
|---|---|---|
| `scripts/backfill_prediction_log.py` (v0.6.0) | False | 脚本一次性回填历史 |
| lifespan startup + scheduler 6h (v0.7.0b) | True | 实时预测, 比赛前自动写 |
| 用户手动调 `record_prediction` | 显式 | 调用方决定 |

### 2.3 两端点

| 端点 | 用途 | 主用 |
|---|---|---|
| `GET /api/elo/live-accuracy?is_live={true/false/null}&model_version=...` | 完整 accuracy stats, 区分数据状态 | 详细分析 |
| `GET /api/elo/live-window-accuracy?days=N` | 近 N 天 mini-card, 永远 live | Cockpit |

返回字段:
```json
{
  "is_live_filter": true | false | null,
  "by_model": {"v3_glicko2": {"samples": 0, "accuracy": null, ...}},
  "overall": {"samples": 0, "accuracy": null, "brier": null, "log_loss": null},
  "data_status": "no_data" | "live_only" | "backfill_only" | "mixed",
  "note": "人类可读提示"
}
```

### 2.4 Cockpit mini-card

位置: Cockpit 顶部, 在 v0.10 freshness 之后
组件:
- 3 列网格: 整体 accuracy / Brier / 数据状态
- 颜色: 准确率 ≥60% 绿 / ≥50% 黄 / <50% 红
- 状态徽章: ⏳ 等待开赛 / ✅ 真 forward / ⚠️ 仅 backfill / 📊 混合
- 端点引用 + note 文本

## 3. 关键边界

| 项 | 处理 |
|---|---|
| 17 天没比赛 | 端点返回 `data_status: backfill_only` + note 解释 |
| backfill 数据怎么算 | 全部历史 1829 样本, is_live=False, accuracy G2 62.7% / Elo 56.7% |
| live 数据何时出现 | scheduler 6h 写预测 + 比赛完赛时 settle_pending_predictions() |
| 比赛开赛后 | is_live=True 行数 = 已完赛比赛数 × 3 模型 |
| 已有 1h dedup | 保留, 不影响 live 字段 |

## 4. 不做 (诚实 push back)

| 项 | 不做原因 |
|---|---|
| 历史 backfill 改 is_live=True | 它们就是 backfill, 改了就违背字段语义 |
| 新增 live 预测预生成 (赛前 7d/3d/1d) | snapshot_group 字段预留, 不实现 |
| 用户触发 live 预测 | 走 lifespan + scheduler 自动, 不暴露手动端点 |
| 真赔率 vs 真 forward 准确率对比 | 留 v0.12 / v0.13 |
| 用户层 (登录收藏) | **不做** (40h+ 无用户) |
