## 🧠 模型对比服务核心

### MODEL_REGISTRY 派发模式
```python
MODEL_REGISTRY = {
    "blend": predict_match_blend,
    "elo": predict_match,
    "glicko2": glicko2_predict_outcome,
}

if model == "blend":
    result = fn(home_code, away_code)  # 不传 db
else:
    result = fn(home_code, away_code, db)

# 统一提取 1X2 概率(blend 返回嵌套)
probs = result.get("probabilities")
if probs is None and "blended" in result:
    probs = result["blended"].get("probabilities")
```

### 价值档位
```python
def _value_tier(rate: float, threshold: float) -> str:
    if rate > 0.10:
        return "strong"   # 强价值,值得考虑(注意 Kelly 比例)
    if rate > threshold:
        return "edge"     # 边缘价值
    return "none"         # 无价值,避开
```

## ⚙️ 零预算 mock 设计

```
1. 客户端启动检查 settings.odds_api_key
   - 有 key → 调真实 API(The Odds API 免费层 500 req/月)
   - 无 key → 自动用 mock(基于 Elo + 5% 利润 + ±2% 噪声)
2. mock 数据来自 DB 已存在的未完赛比赛(kickoff_at.date() == target_date)
3. upsert 时同 (match_id, bookmaker) 覆盖,表只保留最新 1 条
4. 滑动窗口 30 req/min(防止真实 API 免费层被封)
```
