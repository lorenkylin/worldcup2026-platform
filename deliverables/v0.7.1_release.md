## 🆕 新增 API

```
GET /api/simulator/tournament
  ?simulations=10000      # [100, 50000], 默认 10000
  ?model=blend            # blend | elo | glicko2, 默认 blend
  ?return_top_n=8         # top N 对阵频率
  ?seed=42                # 随机种子 (可重现)
```

**响应** (核心字段):
```json
{
  "n_sims": 10000,
  "model": "blend",
  "duration_seconds": 3.97,
  "champion_distribution": {"FRA": 0.131, "ESP": 0.130, "GER": 0.104, ...},
  "finalist_distribution": {...},
  "semifinalist_distribution": {...},
  "quarterfinalist_distribution": {...},
  "r16_distribution": {...},
  "r32_distribution": {...},
  "group_advance_probability": {
    "A": {"MEX": 0.94, "USA": 0.82, "RSA": 0.18, "JPN": 0.05},
    ...
  },
  "top_final_matchups": [
    {"home": "ESP", "away": "FRA", "prob": 0.049, "count": 492},
    ...
  ],
  "top_semifinal_matchups": [...],
  "n_teams": 96,
  "n_groups": 12,
  "total_matches_per_sim": 103
}
```

**语义**: 6 个 "reached" 概率**自然单调递减**
```
group_advance >= r32 >= r16 >= qf >= sf >= finalist >= champion
```

## 🧠 算法核心

### 关键性能优化

| 优化点 | 效果 |
|---|---|
| **48×48 prob_matrix 预计算** | MC 内层只查表,避开 match_prob() 9x9 Poisson 循环 |
| **复用 bracket_logic._assign_third_place_slots** | 8 队第三分配用 v0.3.0 贪心算法,不重写 |
| **二叉树推进** R16→QF→SF | 每轮配对 [0,1] [2,3] ...,无复杂 bracket 追踪 |
| **加时+点球 = 50/50** | 简化:90 分钟平局后两队近似等概率 |

### 公平性

- HOME_BONUS (Elo 50 分) 应用在每场的"主队"
- R32 沿用 FIFA Annex C 槽位
- 已完赛小组赛比分**直接用真实值**,不重打 (更诚实)
- 随机种子可重现 (默认 seed=42)

### 防御性检查

- 同 (home, away) 队配对 → 跳过 (实测触发 0 次)
- R32 槽位解析 None → 跳过 (12 组各 4 队,正常情况不发生)
- SF 决赛 home == away → 跳过 (防御性,正常不发生)

## 🔍 关键 Bug 修复(开发中发现)

### 🐛 Bug 1: R32 槽位重复分配第三名

**症状**: `KeyError: (19, 19)` 在决赛配对时。

**根因**: `_resolve_3()` 让每个 "3XXX" 槽位独立选最佳第三,**未去重**。
结果: A3 同时被 slot 74 (3ABCDF) 和 slot 82 (3AEHIJ) 选中,
r32_winners 出现重复,后续 R16/QF/SF 配对时同队对同队。

**修复**: 改用 `bracket_logic._assign_third_place_slots()` 的贪心分配算法
(v0.3.0 已有,正确实现)。每个 3rd 队伍**只分配到 1 个槽位**。

**测试**: T1 间接覆盖 (集成测试触发后断言会通过)

### 🐛 Bug 2: test seed 与 conftest 冲突

**症状**: `UNIQUE constraint failed: teams.fifa_code`

**根因**: conftest seed MEX(id=1, group=A) + RSA(id=2, group=A),
我的 seed 又加 A1-A4 (group=A),共 6 队混在一组。

**修复**: 在 `_seed_full_worldcup` 开头 `DELETE FROM teams WHERE id IN (1,2)`,
保持测试 12 组 × 4 队 = 48 队干净状态。

### 🐛 Bug 3: 决赛计数语义不一致

**症状**: `F1: finalist(0.0005) < champion(0.016)` 单调性失败

**根因**: `finalist` 计数器只 +1 给 F 输家,语义是"输 F"而非"到达 F"。

**修复**: 重命名 `finalist` = "reached F" (两支决赛队都 +1),输 F 由
`finalist - champion` 推得。6 字段全部按 "P(到达某轮)" 语义统一。

**测试**: T3 + T4 覆盖。
