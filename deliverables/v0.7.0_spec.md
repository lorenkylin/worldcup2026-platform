## 二、v0.7.0 - MarketBlend 校准(2 天)

### 2.1 目标
- **行业标准做法**: 模型 + 市场双轨,用市场赔率反校准模型参数
- **预期收益**: Glicko-2 62.65% → ≥65.65%(+3pp)
- **行业案例**: Pinnacle / Betfair / Kaggle 公开方案均证实此方法有效

### 2.2 三个子任务

#### 0.7.0a prediction_log 自动写库(0.5 天)
- lifespan startup 触发 + 6h odds 刷新后调度
- 复用 `app/services/prediction_log.py` 已有的 `write_log` 接口
- 实盘预测 = 自动累积准确率统计样本
- **回填与实盘统一数据源** → 准确率统计持续滚雪球

#### 0.7.0b 市场反校准(1 天)
- 新增 `app/services/calibration.py`
- 用 Pinnacle/wc26 odds(去 vig)作为"软 ground truth"
- Platt scaling 训练:基于 v0.6 已有 1826 行 walk-forward 数据
- 算法:
  ```
  模型原始 p_home / p_draw / p_away
       ↓
  用市场隐含概率(p_market)做回归
       ↓
  学习修正系数 a, b: p_calibrated = sigmoid(a * logit(p_model) + b)
       ↓
  Blend: p_final = α * p_calibrated + (1-α) * p_market
  ```
- 输出 Blend 概率 + 不确定性区间

#### 0.7.0c 双轨 dashboard(0.5 天)
- 新增 `/api/elo/market-blend` 端点
- `/accuracy` 页加 4 列表(Elo / Glicko-2 / Market / Blend)
- 复用 v0.6 3 模型对比表结构,加 1 列即可
- 加"校准日期"提示,季度重训提醒

### 2.3 风险 & 缓解
| 风险 | 等级 | 缓解 |
|---|---|---|
| 样本量 | 🟢 | 1826 行 walk-forward 数据够用 |
| 市场噪声 | 🟡 | 7 天滑动平均 + Pinnacle 优先 |
| 校准漂移 | 🟡 | `last_calibrated_at` 字段 + 季度重训 |
| 模型 vs 市场冲突 | 🟡 | α=0.6 (模型) / 0.4 (市场) 经验值,后续调 |

### 2.4 不做
- 不引入 Pinnacle 付费 API
- 不重写 v0.5 odds 端点
- 不引入 PyMC / Stan 复杂贝叶斯(用 sklearn LogisticRegression 即可)

## 三、v0.7.1 - Simulator 整届 Monte Carlo(1-2 天)

### 3.1 目标
- 单场 → 整届 64 场,10000 次模拟
- 输出: 每队夺冠概率 + 各阶段概率(16强/8强/4强/决赛/夺冠)
- **复刻 FiveThirtyEight / ESPN BPI 风格的 tournament predictor**

### 3.2 三个子任务

#### 0.7.1a 比赛树生成器(0.5 天)
- 复用 v0.3.0 bracket 真实小组赛排名 + 8 最佳第三
- 输出 64 场对阵(R32 + R16 + QF + SF + F)
- 按 FIFA 官方对阵规则(2A vs 2B 等)
- 小组赛未结束 → fallback 用 v0.3.1 的 bracket service

#### 0.7.1b Monte Carlo 引擎(0.5 天)
- 复用 `app/services/simulator.py` 单场逻辑(已存在)
- 新增 `app/services/monte_carlo.py`:
  ```python
  async def simulate_tournament(times: int = 10000) -> dict:
      # 进度跟踪
      # 10000 次并行(asyncio.gather + ProcessPoolExecutor)
      # 累计每队晋级次数
      # 输出: {team_id: {r32: 1.0, r16: 0.85, qf: 0.6, ...}}
  ```
- 性能目标: <5s / 10000 次
- Caching: 球队 ELO 不变 → 结果 30min cache

#### 0.7.1c 前端接入(0.5 天)
- `/#/bracket` 加"夺冠概率"列
- 新增 `/#/simulator` 整届页
- Chart.js 横向柱状图(48 队)
- 移动端 375px 友好

### 3.3 风险 & 缓解
| 风险 | 等级 | 缓解 |
|---|---|---|
| 性能 10000×64=64万 | 🟡 | ProcessPoolExecutor + caching |
| R32 8 最佳第三对阵 | 🟡 | FIFA 官方文档 + 简化为 12 组对应 |
| 冷启动 | 🟡 | 小组赛未结束 → fallback v0.3.1 |

### 3.4 不做
- 不接入实时比分(bracket 用预测)
- 不做小组赛 Monte Carlo(只 R32 起,减少 95% 计算量)
- 不做球员级蒙特卡洛(无球员数据)

## 四、v0.7.2 - 数据源扩展(1 周)

### 4.1 目标
- 加 FIFA 官方赛事数据(无球员级)
- 加 Transfermarkt 联赛数据(无球员级)
- 集成到现有 wc26 + fb-data 备份链路
- **0 schema 改动** ✅(重要,避免回填地狱)

### 4.2 三个子任务

#### 0.7.2a FIFA 官方集成(2 天)
- 调研: 申请合作伙伴 API / RSS fallback
- 集成: 比赛元数据 / 场馆 / 裁判 / 天气
- 不存数据库(纯 HTTP 集成),只在响应里附加元数据
- 缓存 24h

#### 0.7.2b Transfermarkt 联赛数据(2 天)
- 爬虫(无官方 API)
- 数据: 联赛排名 / 进球 / 失球 / 阵容名单(国家维度)
- 合规: User-Agent + 限速 10 req/min + 缓存 7 天
- 不爬球员个人页(避免法律风险,留给 v0.7.3)

#### 0.7.2c 数据融合 + 三路 fallback(1-2 天)
- 新增 `app/services/external_data.py`
- 优先级: wc26 (主,15min 实时) > FIFA 官方 > Transfermarkt
- 前端显示数据源徽章:"wc26" / "FIFA 官方" / "Transfermarkt"
- Admin 触发手动同步

### 4.3 风险 & 缓解
| 风险 | 等级 | 缓解 |
|---|---|---|
| Transfermarkt 法律 | 🟡 | 公开数据 + 限速 10 req/min + 缓存 |
| FIFA API 权限 | 🟡 | 不申请到则 RSS fallback |
| 数据冲突 | 🟡 | 优先级排序 + 时间戳版本 |
| 爬虫稳定性 | 🟡 | 重试 + 指数退避 + Mock 注入 |

### 4.4 不做(诚实 push back)
- **球员级数据**(推 v0.7.3)
- **阵容影响模型**(需球员数据)
- **新 schema migration**(本版本 0 schema 改动)
- **球员身价/转会**(敏感数据,推下个 sprint)
