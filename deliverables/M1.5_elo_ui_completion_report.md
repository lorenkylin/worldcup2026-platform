1. 默认显示 Top 1 (ESP) vs 末位 (GUA)，直观展示"极端对决"

## 4. 关键技术细节

### 4.1 `strengthScore(elo, topElo)` — 实力分计算

```js
function strengthScore(elo, topElo) {
  if (!topElo || topElo < 1400) return 0;
  // 锚定 1400 作为"最低"参考，Top 1 = 100
  return Math.max(0, Math.min(100, ((elo - 1400) / (topElo - 1400)) * 100));
}
```

- **参考线 1400**：略低于 1500（国际平均），保证 GUA (1416) 也能拿到 ~2.6 分而非 0
- **Top 1 = 100**：保证 ESP (2010) = 100%，视觉上 Top 1 "满条"
- **clamp 0-100**：避免负值或 > 100

### 4.2 三色概率条

```js
const hWin = (p.probabilities.home_win * 100).toFixed(1);
const draw = (p.probabilities.draw * 100).toFixed(1);
const aWin = (p.probabilities.away_win * 100).toFixed(1);
```

宽度用百分比，不需要额外算，flex 自然按 sum=100% 排。

### 4.3 智能结论

```js
const maxProb = Math.max(p.probabilities.home_win, p.probabilities.draw, p.probabilities.away_win);
const winnerText = maxProb === p.probabilities.home_win ? '主胜倾向' 
  : maxProb === p.probabilities.away_win ? '客胜倾向' 
  : '平局倾向';
```

按最大概率项给结论，user 不用心算。

### 4.4 select 联动

```js
setTimeout(() => {
  const update = async () => { ... };
  homeSel.addEventListener('change', update);
  awaySel.addEventListener('change', update);
}, 50);
```

`setTimeout(50)` 等待 DOM 渲染（innerHTML 异步），然后绑事件。

## 8. 下一步建议（M1.6 候选）

| 候选 | 内容 | ROI |
|---|---|---|
| 冠军概率 | 1000 次 Monte Carlo tournament sim，给每队夺冠% | 中（用户可见，但前端需新算法） |
| 实力分升级 | 用 logistic 胜率 vs 平均 1500 而非简单归一化 | 低（视觉差异小） |
| Elo 走势图 | 选队后展示近 1 年 Elo 变化曲线 | 中（需 DB 增量） |
| 预测准确率追踪 | 比赛结束后自动回填 actual_score + 算 Brier | 高（数据驱动优化） |

按主人节奏推进，建议**冠军概率**作为 M1.6，因为它与 P1.1（赛后复盘）天然互补。
