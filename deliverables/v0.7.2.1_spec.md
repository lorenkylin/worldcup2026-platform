# v0.7.2.1 前端赔率接入 Spec

> 目标:让主人和用户在浏览器中真正看到 v0.7.2 的 4 个赔率端点结果,而不是只在 curl / E2E 里。
> 范围:仅前端改动,不动后端。

---

## 1. 当前状态

`/#/odds` 页面已存在(v0.5.1 实现),目前展示:
- 单场比赛赔率卡(`oddsLatest`)
- 多公司赔率对比(`oddsCompare`)
- 价值投注列表(`valueBets`,基于 `valueBetEdge ≥ 5%`)

但 v0.7.2 新增的 3 个端点**前端未接入**:

| 端点 | 当前页面状态 |
|---|---|
| `GET /api/odds/compare-model?match_id=&model=` | ❌ 未接入 |
| `GET /api/odds/value-bets-model?model=&min_tier=` | ❌ 未接入 |
| `GET /api/odds/service-status` | ❌ 未接入(只能看后台 curl) |

---

## 2. 设计假设

1. **不重做页面** — 复用现有 `/#/odds` 页面和样式,加 1 个新 section + 2 处增强。
2. **不破坏 v0.5.1 旧 API** — 旧的 `/odds/compare` 和 `/odds/value-bets` 保留,作为 fallback。
3. **保持主人"先说如何做再动手"流程** — 本 spec 先出,主人确认后奴才动代码。
4. **模型选择器常驻** — 3 个模型(elo/glicko2/blend)始终可选,默认 blend(与 MC 默认一致)。

---

## 3. UI 改动(3 处)

### 3.1 新增"服务状态"小卡(页面顶部)
- 显示当前 `provider` (mock/the-odds-api) + `rate_limit_remaining` + `last_fetch_at`
- 用一个紧凑的横条,左边蓝点表示在线、绿色对勾表示 API 正常
- 数据来源:`GET /api/odds/service-status`

### 3.2 现有赔率卡升级:模型选择器
- 在每场比赛赔率卡右上角加一个 `<select>` 下拉:`模型: [Elo|Glicko-2|Blend]`
- 选中后异步 fetch `/api/odds/compare-model?match_id=&model=` 替换卡内的模型概率
- 默认 `Blend`,与 Cockpit/MC 默认一致

### 3.3 新增"按模型找价值投注"section(页面下半)
- 现有"价值投注"section 之前,插入一个新 section:`🎯 按模型筛选价值投注`
- 控件:`模型 [Blend]` + `最低价值 [高价值 | 中价值 | 任意]`
- 数据:`GET /api/odds/value-bets-model?model=blend&min_tier=edge`
- 列表展示:match + 模型概率 + 市场隐含 + edge% + 推荐投注
- 已有价值投注 section 不删除(v0.5.1 老用户)

## 7. 风险与兜底

1. **fetch 失败** — `loadOddsServiceStatus` 失败时横条显示"⚠️ 服务状态不可用",不阻塞页面渲染。
2. **模型切换竞态** — 快速切模型时,加 `req_seq` 防止旧响应覆盖新结果(标准模式)。
3. **数据为空** — `value-bets-model` 返回空时显示"暂无该模型/阈值下的价值投注"。
