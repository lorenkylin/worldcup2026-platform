# v0.4.0 StatsBomb Elo 可行性评估与方案选择

> 评估时间：2026-06-15
> 评估范围：StatsBomb Open Data 用于替代/补充当前 Hicruben Elo 评分体系的可行性

---

## 一、核心发现（必须先说）

### 1.1 StatsBomb Open Data 国际赛事覆盖有限

| 赛事 | 赛季 | 场数 | 说明 |
|:---|:---|---:|:---|
| FIFA World Cup | 2018 | 64 | 完整 |
| FIFA World Cup | 2022 | 64 | 完整 |
| UEFA Euro | 2020 | 51 | 完整 |
| UEFA Euro | 2024 | 51 | 完整 |
| Copa América | 2024 | 32 | 完整 |
| Africa Cup of Nations | 2023 | 52 | 完整 |
| **合计** | — | **~314** | **仅国际赛** |

对比当前 Hicruben 数据：
- **Hicruben**: 913 场（2023-11 ~ 2026-06），覆盖友谊赛、预选赛、洲际赛
- **StatsBomb Open Data**: ~314 场，**缺少 2023-2026 的友谊赛、预选赛、国家联赛**
- **2026 世界杯尚未举办**，StatsBomb 没有任何 2026 世界杯数据

### 1.2 2026 参赛队覆盖不完整

基于现有 48 队 DB 与 StatsBomb 国际赛事球队名单比对：

**明确缺失的球队（至少 4-8 支）**：
- `IRQ` 伊拉克 — 未出现在任何 StatsBomb 国际赛事中
- `UZB` 乌兹别克斯坦 — 未出现
- `JOR` 约旦 — 未出现
- `NZL` 新西兰 — 未出现在这些赛事中
- `BIH` 波黑 — 未出现
- `CUW` 库拉索 — 未出现
- `HAI` 海地 — 未出现

**队名映射风险**：
- `CPV` → StatsBomb 写作 "Cape Verde Islands"（非 "Cape Verde"）
- `COD` → StatsBomb 写作 "Congo DR"（DB 可能是 "DR Congo"）
- `IRN` → StatsBomb 写作 "Iran"（正常）
- `KOR` → StatsBomb 写作 "South Korea"（正常）
- `USA` → StatsBomb 写作 "United States"（正常）

### 1.3 数据时效性问题严重

StatsBomb Open Data 国际赛最新只到 **2024 年中**（Copa América 2024 / Euro 2024）。

当前平台预测的赛事是 **2026 世界杯**，时间差：
- 友谊赛窗口：2024-06 到 2026-06 共约 24 个月
- 预选赛：大部分大洲 2024-2025 年进行
- **这些关键状态变化，StatsBomb Open Data 完全缺失**

### 1.4 许可证要求

StatsBomb Open Data 条款：
> "If you publish, share or distribute any research, analysis or insights based on this data, please state the data source as StatsBomb and use our logo."

- 允许研究和公开使用
- **必须标注 StatsBomb 并使用其 logo**
- 未明确禁止预测/博彩用途，但商业使用需关注品牌归属

## 二、诚实的结论

### ❌ 不建议把 StatsBomb 作为 Elo 主数据源

原因：
1. **数据量不足**：314 场 vs 913 场，模型训练样本少 66%
2. **时效性差**：缺少 2024-2026 最关键的状态变化
3. **覆盖不全**：至少 4-8 支 2026 参赛队无数据
4. **无近期友谊赛/预选赛**：Elo 模型最依赖的定期更新数据缺失

**量化风险**：用 StatsBomb 重新训练 Elo，预计回测准确率大概率低于当前 Hicruben 的 58.3%，且对亚洲/非洲球队预测严重偏差。

### ✅ StatsBomb 可以作为补充/对比数据源

价值：
1. **事件级数据质量高**：有 xG、射门、传球等字段，可用于更精细的特征工程
2. **大赛数据权威**：世界杯/欧洲杯/美洲杯/非洲杯的比赛结果可靠
3. **适合验证 Hicruben 模型**：用同一套比赛对比两套 Elo 的差异

## 三、可选方案

### 方案 A：StatsBomb 作为对比数据源（推荐）

**实现**：
- 下载 StatsBomb 国际赛结果
- 建立 `SB_NAME_TO_FIFA` 映射
- 用同样 Elo 算法（K=60, home_bonus=70）重新训练一套评分
- 在现有 `/api/elo` 中新增 `source` 参数：`hicruben`（默认）/ `statsbomb`
- 对无数据的球队 fallback 到 Hicruben

**优点**：
- 不影响现有主模型
- 用户可对比两套评分
- 工作量可控（~2-3 天）
- 规避数据缺失风险

**缺点**：
- StatsBomb 评分本身质量可能不如 Hicruben
- 需要额外维护一套映射

### 方案 B：混合模型（Hicruben 为主 + StatsBomb xG 增强）

**实现**：
- Hicruben 维持主 Elo
- 对 StatsBomb 有数据的球队，用 xG 差异计算一个"表现调整因子"
- 例如：某队实际进球 < xG，说明运气差，Elo 不降太多
- 仅用于有大赛数据的球队

**优点**：
- 保留 Hicruben 的覆盖和时效
- 引入 StatsBomb 的高质量事件数据

**缺点**：
- 模型复杂度显著增加
- 需要 careful 验证，否则容易过拟合

### 方案 C：放弃 StatsBomb Elo，改为 StatsBomb 事件数据展示

**实现**：
- 不重新训练 Elo
- 在 `/api/matches/{id}/stats` 或新增 `/api/matches/{id}/xg` 中展示 StatsBomb 的 xG 数据
- 仅针对 2018/2022 世界杯等历史比赛

**优点**：
- 最轻量
- 数据质量高、可视化价值大
- 无模型风险

**缺点**：
- 不是用户原来说的 "StatsBomb Elo"
- 对 2026 预测无直接帮助

### 方案 D：坚持做纯 StatsBomb Elo 替代

**实现**：
- 用 StatsBomb ~314 场国际赛训练新 Elo
- 缺失球队用 1500 初始分或 Hicruben 分数填充
- 完全替换 Hicruben 数据源

**风险**：
- 覆盖不全、时效性差
- 预测质量大概率下降
- **不推荐**

---

## 四、建议决策

**推荐方案 A**：StatsBomb 作为可切换的对比数据源，Hicruben 保持默认主数据源。

理由：
1. 风险最低，不影响现有 58.3% 准确率的模型
2. 实现成本可控
3. 满足 "接入 StatsBomb" 的目标
4. 为后续方案 B（xG 增强）留出接口

---

## 五、若选择方案 A 的实现路径

| 步骤 | 工作 |
|:---|:---|
| A1 | 下载 StatsBomb 6 项国际赛 match 文件 |
| A2 | 解析并标准化比赛结果（home/away/score/date） |
| A3 | 建立 `SB_NAME_TO_FIFA` 映射表（处理 Cape Verde Islands / Congo DR 等） |
| A4 | 用现有 `elo_update()` 重新训练 Elo，输出 `statsbomb_elo.json` |
| A5 | 修改 `app/services/elo.py`：支持 `source` 参数 |
| A6 | 修改 `/api/elo` 相关端点：默认 `hicruben`，可选 `statsbomb` |
| A7 | 缺失球队 fallback 到 Hicruben，并在响应中标记 |
| A8 | 新增 `/api/elo/compare/{home}/{away}`：同时返回两套预测 |
| A9 | pytest + Playwright 验证 |
| A10 | README + MEMORY.md 更新，版本 bump 至 v0.4.0 |

预计工作量：**1 个完整迭代**（比 Bracket 复杂，因为涉及数据下载、映射、训练）。
