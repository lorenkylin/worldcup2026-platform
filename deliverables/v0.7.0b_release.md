## 🆕 新增服务

### `auto_log_predictions(db, models=("v1_elo","v3_glicko2","v7a_blend"), lookahead_days=7)`

**职责**: 扫描未来 7 天未完赛比赛 → 调对应模型 predict → 写 `prediction_log`(1h dedup)

**返回**:
```python
{
  "matches_scanned": N,
  "predictions_added": K,
  "predictions_skipped": M,
  "by_model": {"v1_elo": a, "v3_glicko2": b, "v7a_blend": c},
  "errors": []
}
```

**契约**:
- ✅ 1h dedup:同 (match_id, model) 1 小时内只写 1 条
- ✅ 已完赛 (有比分) 跳过
- ✅ 窗口外 (默认 7 天) 跳过
- ✅ 未知球队 → 3 个 predict_fn 都返回 None → 不写库,不报错
- ✅ 单条错误隔离:`errors` 列表收集但不中断循环
- ✅ 子集模型:`models=("v1_elo", "v7a_blend")` 可只写指定

### `run_periodic_refresh` Step 3 (6h 周期)

`step 3: predictions` 追加到现有 odds snapshot / fb-data 之后,返回 `predictions_added` / `predictions_by_model`。

## 📌 关键工程教训

1. **lifespan startup 立即跑业务** 比等 15min scheduler 第一次轮询好(避免数据 stale 窗口)
2. **测试用 match_id=88888** 隔离 dedup(生产不存在的 ID)
3. **lifespan 写库失败隔离**:单条 error 不中断循环
4. **3 模型注册表模式** `MODEL_REGISTRY` 字典比 if/elif 链好扩展
5. **JS 重渲整页** vs 局部 diff:UI 简单时前者更稳
