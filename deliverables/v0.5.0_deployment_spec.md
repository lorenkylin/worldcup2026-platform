# v0.5.0 全栈部署方案（方案 A）

> 版本：v0.5.0（部署阶段）
> 日期：2026-06-15
> 状态：待用户确认

---

## 一、需求确认

用户选择：
- **部署形式**：完整前后端上线
- **平台偏好**：腾讯云 / EdgeOne Pages
- **运行要求**：持续运行 + 定时同步（15min worldcup26.ir + 30min form 回填 + Bracket 自动重算）

**关键判断**：由于项目使用 **FastAPI + SQLite + APScheduler 后台任务**，EdgeOne Pages（Serverless/边缘托管）不适合承载有状态数据库和常驻后台任务。推荐改用 **腾讯云轻量应用服务器 / CVM** 部署完整应用。

---

## 二、推荐架构

```
用户
  ↓ HTTPS (443)
域名（如 wc2026.example.com）
  ↓
腾讯云 Lighthouse / CVM
  ├── Nginx（反向代理 + SSL）
  │   ├── /api/* → uvicorn (127.0.0.1:8000)
  │   └── /*     → static files
  ├── uvicorn + FastAPI（systemd 托管）
  │   ├── SQLite: /opt/wc2026/data/worldcup2026.db
  │   └── APScheduler: 15min 同步 + 30min form + Bracket 自动重算
  └── Python venv
```

| Admin Token | 是 | `app/config.py` 中 `ADMIN_TOKEN`，部署后需保密 |

## 四、部署步骤

### 4.1 服务器准备

1. 购买腾讯云 Lighthouse（推荐 Ubuntu 22.04 LTS，2C2G/2C4G，北京/上海节点）
2. 开放安全组端口：22（SSH）、80（HTTP）、443（HTTPS）
3. 配置域名 A 记录指向服务器公网 IP

### 4.2 环境安装

```bash
# 连接服务器
ssh ubuntu@<服务器公网IP>

# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装依赖
sudo apt install -y python3 python3-venv python3-pip nginx git certbot python3-certbot-nginx

# 创建应用目录
sudo mkdir -p /opt/wc2026
sudo chown ubuntu:ubuntu /opt/wc2026
cd /opt/wc2026

# 拉取代码（或上传）
git clone <你的仓库地址> .
# 或从本地 scp 上传

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 4.3 初始化数据

```bash
cd /opt/wc2026
python data/seed.py
```

### 4.4 配置 Systemd 服务

创建 `/etc/systemd/system/wc2026.service`：

```ini
[Unit]
Description=2026 FIFA World Cup Analysis Platform
After=network.target

[Service]
Type=simple
User=ubuntu
Group=ubuntu
WorkingDirectory=/opt/wc2026
Environment="PATH=/opt/wc2026/venv/bin"
Environment="PYTHONPATH=/opt/wc2026"
Environment="ADMIN_TOKEN=你的管理员Token"
ExecStart=/opt/wc2026/venv/bin/python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启用并启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable wc2026
sudo systemctl start wc2026
```

### 4.5 配置 Nginx

创建 `/etc/nginx/sites-available/wc2026`：

```nginx
server {
    listen 80;
    server_name wc2026.example.com;

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location / {
        root /opt/wc2026/app/static;
        try_files $uri $uri/ /index.html;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/wc2026 /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 4.6 配置 SSL（Let's Encrypt）

```bash
sudo certbot --nginx -d wc2026.example.com
```

## 六、维护与监控

### 6.1 常用命令

```bash
# 查看服务状态
sudo systemctl status wc2026

# 查看日志
sudo journalctl -u wc2026 -f

# 重启服务
sudo systemctl restart wc2026

# 手动触发同步
curl -H "X-Admin-Token: 你的Token" https://wc2026.example.com/api/admin-sync/worldcup26/full
```

### 6.2 监控项

- `/health` 是否返回 200
- 服务器 CPU / 内存 / 磁盘
- worldcup26.ir 同步是否成功（看日志）
- Nginx 访问日志中的 5xx 错误

## 七、回滚策略

| 场景 | 回滚方式 |
|---|---|
| 新版本 bug | `git checkout <上一个tag>` + `sudo systemctl restart wc2026` |
| 服务无法启动 | `sudo systemctl stop wc2026` + 回退代码 + 重启 |
| 数据库损坏 | 恢复备份（需提前配置定时备份）|
| 严重故障 | 关闭 443 端口或回退 DNS 解析 |

## 八、更快但受限的替代：静态快照

如果只是想先给别人看看效果、不追求实时同步，可以：
- 用当前数据生成一组静态 HTML 页面
- 部署到 EdgeOne Pages
- 结果：免费/低价、访问快，但数据不会自动更新

此方案不满足你选择的“持续运行 + 定时同步”，仅作为临时演示备选。
