### 4. `_temp_db` fixture 大优化

**原实现**：每个测试都执行 `create_engine → drop_all → create_all → seed`，导致 400+ 测试累积 ~7 min。  
**新实现**：

- 进程首次需要时生成一份**模板 SQLite 文件**（含 schema + 最少种子数据）；
- 每个测试通过 `shutil.copy` 拷贝模板文件，再创建 engine/session；
- 进程退出时 `atexit` 删除模板文件。

效果：核心测试从 **~7 min 降到 ~16 s**，提升约 **25 倍**。

## 三、关键说明

1. **E2E 在 CI 中的数据**：
   - E2E job 先执行 `python data/seed.py`，因此 `BRA`/`ARG` 等队存在，与本地一致。
   - 赔率相关 E2E 使用默认 `odds_api_provider = mock`，无需真实 API key。

2. **Admin Token**：
   - CI 未配置 `ADMIN_TOKEN` secret 时，会自动使用兜底 `ci-admin-token`。
   - 若主人在 GitHub Secrets 设置了 `ADMIN_TOKEN`，则以设置值为准。

3. **模板 DB 隔离性**：
   - 每个测试仍拿到独立的物理 SQLite 文件，测试间数据完全隔离。
   - 测试内如有 `commit()`，只影响该测试自己的拷贝，不会污染模板。
