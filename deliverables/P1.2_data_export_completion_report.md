3. **新增 `csvEscape()` 工具函数**（6 行 RFC 4180 转义）：

```js
function csvEscape(value) {
  // RFC 4180: 含逗号/双引号/换行的字段加双引号；内部双引号转义为两个双引号
  const s = String(value == null ? '' : value);
  if (/[",\r\n]/.test(s)) {
    return '"' + s.replace(/"/g, '""') + '"';
  }
  return s;
}
```

## 6. 关键经验沉淀

### 6.1 Python `read_text()` 的 universal newline 陷阱

Python `pathlib.Path.read_text()` 默认开启 universal newline 模式，会自动把 `\r\n` 规范化成 `\n`。验证 CSV 行结束符必须用 `read_bytes()`：

```python
raw_bytes = local_path.read_bytes()  # 不规范化
assert raw_bytes.startswith(b'\xef\xbb\xbf'), '缺 UTF-8 BOM'
content = raw_bytes[3:].decode('utf-8')  # 剥 BOM
lines = content.split('\r\n')  # 正确按 CRLF 切分
```

教训：**验证二进制文件格式时永远用 `read_bytes()`**。

### 6.2 Playwright `page.evaluate` 参数的 JSON 序列化陷阱

```python
# ❌ 错误：Python 字符串含特殊字符（逗号/引号/中文）经过 JSON 序列化会损坏
escaped = await page.evaluate(
    '([parts]) => csvEscape(parts[0])',
    ['逗号,字段', '含"引号', '普通']
)
# 结果：CSV 输出"逗,号,","",普通" —— 完全乱了

# ✅ 正确：把测试数据写死在 JS 字符串里，不经过 Python-JSON 边界
escaped = await page.evaluate('''() => {
    return csvEscape('逗号,字段') + '|' + csvEscape('含"引号') + '|' + csvEscape('普通');
}''')
```

教训：**Playwright 跨语言参数传递只传简单标量（数字/布尔），含特殊字符的字符串直接写在 JS 里**。

### 6.3 UTF-8 BOM 对 Excel 中文版至关重要

不加 BOM：
- Excel 中文版打开 CSV → 中文乱码（按 GBK 解码）
- macOS Numbers / Google Sheets → 中文正常

加 BOM：
- Excel 中文版 → 中文正常
- 其他工具 → 也正常（UTF-8 BOM 是 Unicode 标准）

实现：文件开头加 `\uFEFF`（U+FEFF REVERSED QUESTION MARK 等同 ZERO WIDTH NO-BREAK SPACE）。

### 6.4 RFC 4180 转义三条规则

1. 字段含 `,` 或 `"` 或 `\r` 或 `\n` → 整个字段加双引号
2. 字段内部的 `"` → 替换为 `""`
3. 行结束符 → CRLF（`\r\n`）

测试覆盖三条规则的最小样例：
```js
csvEscape('逗号,字段')  // → "逗号,字段"
csvEscape('含"引号')    // → "含""引号"
csvEscape('普通')        // → 普通
```
