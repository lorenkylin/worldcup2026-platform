# v0.4.0 Spec：StatsBomb Elo 作为对比数据源

> 决策：方案 A —— StatsBomb 作为可切换对比数据源，Hicruben 保持默认主数据源。
> 版本：v0.4.0
> 日期：2026-06-15

---

## 1. Objective

在现有 Hicruben Elo 模型基础上，新增 StatsBomb Open Data 训练的第二套 Elo 评分，使用户/开发者能够：

1. 通过 `source` 参数切换数据源（默认 `hicruben`，可选 `statsbomb`）
2. 通过 `/api/elo/compare/{home}/{away}` 同时对比两套评分对同一场比赛的预测
3. 缺失 StatsBomb 数据的球队自动 fallback 到 Hicruben，并在响应中标记

成功标准：
- 所有 `/api/elo/*` 端点支持 `?source=statsbomb`
- 新增 `/api/elo/compare/{home}/{away}` 端点
- StatsBomb Elo 覆盖至少 40/48 支 2026 参赛队
- 缺失球队 fallback 到 Hicruben，不报错
- pytest ≥8 项新增测试全绿，Playwright E2E ≥1 项
- README 和 MEMORY.md 更新

## 5. API Contract

### 5.1 现有端点扩展

所有以下端点新增可选 query 参数 `?source=statsbomb`（默认 `hicruben`）：

- `GET /api/elo/ratings?source=statsbomb`
- `GET /api/elo/ratings/{fifa_code}?source=statsbomb`
- `GET /api/elo/top?limit=10&source=statsbomb`
- `GET /api/elo/predict/{home}/{away}?source=statsbomb`
- `GET /api/elo/predict-enhanced/{home}/{away}?source=statsbomb`
- `GET /api/elo/backtest`（保持 hicruben 专用，或返回双源 backtest）

### 5.2 新增端点

```
GET /api/elo/compare/{home_code}/{away_code}
```

返回示例：

```json
{
  "home": {"fifa_code": "ARG", "name_zh": "阿根廷"},
  "away": {"fifa_code": "BRA", "name_zh": "巴西"},
  "hicruben": {
    "home_elo": 1976,
    "away_elo": 1955,
    "probabilities": {"home_win": 0.38, "draw": 0.28, "away_win": 0.34},
    "expected_goals": {"home": 1.35, "away": 1.25},
    "data_as_of": "2026-06-11T16:47:05.772Z"
  },
  "statsbomb": {
    "home_elo": 1890,
    "away_elo": 1920,
    "probabilities": {"home_win": 0.34, "draw": 0.29, "away_win": 0.37},
    "expected_goals": {"home": 1.28, "away": 1.32},
    "data_as_of": "2024-07-15T00:00:00.000Z",
    "rating_source": {"home": "statsbomb", "away": "statsbomb"}
  },
  "model": "elo_dixon_coles_v1",
  "attribution": "StatsBomb data provided by StatsBomb. Used under open data terms."
}
```

对于 StatsBomb 缺失的球队，例如 `IRQ`：

```json
{
  "statsbomb": {
    "home_elo": 1520,
    "away_elo": 1955,
    "rating_source": {"home": "hicruben_fallback", "away": "statsbomb"},
    "fallback_reason": {"home": "Team IRQ not found in StatsBomb data"}
  }
}
```

## 6. Algorithm

### 6.1 Elo 训练

复用现有 `app/services/elo.py` 中的 `elo_update(rating_a, rating_b, score_a, home_bonus_a=0, k=K_FACTOR_WC)`。

步骤：
1. 合并 6 个比赛的 match 文件
2. 按 `match_date` 升序排序
3. 每队初始分 1500
4. 对每场比赛：
   - 主队：`score = 1` if 主队胜，`0.5` if 平，`0` if 负
   - `home_bonus = 0`（大赛均为中立场，避免"名单主队"偏差）
   - `k = K_FACTOR_WC = 60`
   - 更新双方 Elo
5. 输出 `data/seed/statsbomb/statsbomb_elo.json`：
   - `ratings`: {fifa_code: rating}
   - `generatedAt`: ISO 时间
   - `matchesApplied`: 314
   - `sourceCompetitions`: 6 项赛事列表
   - `missingTeams`: 缺失球队列表

### 6.2 Team Name Mapping

创建 `SB_NAME_TO_FIFA`：

```python
{
    "Argentina": "ARG",
    "Brazil": "BRA",
    "United States": "USA",
    "South Korea": "KOR",
    "Iran": "IRN",
    "Cape Verde Islands": "CPV",
    "Congo DR": "COD",
    # ... 共覆盖 StatsBomb 国际赛中出现的所有球队
}
```

### 6.3 Fallback 策略

```python
def get_team_elo_statsbomb(fifa_code: str) -> Optional[int]:
    rating = _statsbomb_ratings.get(fifa_code.upper())
    if rating is not None:
        return rating, "statsbomb"
    # fallback
    hicruben = get_team_elo(fifa_code)
    return hicruben, "hicruben_fallback"
```

## 9. Boundaries

- **Always do**: 所有新端点返回完整 `data_source` / `rating_source` 标记；缺失球队 fallback 不报错
- **Ask first**: 是否把 StatsBomb 评分设为默认；是否在首页展示 StatsBomb logo
- **Never do**: 删除 Hicruben 数据；让 StatsBomb 缺失球队返回 null/报错；把未授权原始数据公开到前端

## 11. Risks & Mitigations

| Risk | 影响 | 缓解 |
|:---|:---|:---|
| StatsBomb 下载失败 | 无法生成 Elo | 脚本支持重试；提交生成的 JSON 文件到 repo 作为备份 |
| 队名映射遗漏 | 某些球队无法识别 | 建立未知名称白名单；在生成时打印警告 |
| 评分与 Hicruben 不可比 | 用户困惑 | compare API 明确标注数据时间；前端并排展示 |
| 许可证合规 | 法律风险 | `attribution.md` + API 返回 attribution 字段 |
