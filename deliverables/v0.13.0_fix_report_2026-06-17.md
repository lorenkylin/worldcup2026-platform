## 二、关键修复详情

### 2.1 `/api/odds/compare-model` 模型分发修复

**原代码**：`elo`/`glicko2` 统一把 `db: Session` 当第三个参数传入，导致：
- `elo`：Session 被当 `source`，静默回退 hicruben
- `glicko2`：直接 500

**修复后**：按模型显式组装参数
- `elo` → `predict_match(home, away)`
- `glicko2` → 先 `lookup_glicko2_rating`，再 `predict_outcome(rating, rd, rating, rd, home_bonus=HOME_BONUS)`
- `blend` → `predict_match_blend(home, away)`

**新增回归测试**：`test_compare_match_odds_elo_model` / `test_compare_match_odds_glicko2_model`

### 2.2 Fly.io 部署链修复

- **移除 `release_command: "alembic upgrade head"`**：Fly release_command 在临时机运行，不挂载 `/data` 持久卷，对 SQLite 无效且危险。
- **`APP_DEBUG` → `DEBUG`**：与 `config.py` 字段名一致，避免生产环境仍以 `debug=true` 运行。
- 更新 `test_v013_fly.py` 断言，确保 `release_command` 不含 alembic。

### 2.3 前端关键 bug 修复

- **`/#/health` 404**：`apiWithRetry('/api/health/sources')` → `apiWithRetry('/health/sources')`
- **Bracket 跳转错误**：后端 `/api/bracket` 返回真实 `match.id`；前端 `linkHref` 改用 `m.id`
- **重试闭包丢失**：引入 `window.__retryFns` 注册表 + `window.__runRetry(id)`，避免函数字符串化

### 2.4 数据层修复

- 新增 Alembic 迁移 `ccd9db6f49a1`：补 `prediction_log.is_live` 索引
- 新增 Alembic 迁移 `e1bc3cd68e68`：删除孤儿表 `odds_api_cache`
- 本地 SQLite 已手动应用上述变更
