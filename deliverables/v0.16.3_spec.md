# v0.16.3 Spec — 推荐比分 UI 醒目化

> 2026-06-24
> **目标：在 v0.16.2 基础上提升推荐比分的视觉权重，让用户在深色玻璃拟态界面中一眼捕捉首选/次选比分。**

## 1. 背景

v0.16.2 已经把 `primary_score` / `secondary_score` 展示到前端三处核心位置，但初始样式较为朴素（白色或 slate 文字），在深空背景下不够突出。v0.16.3 通过 amber/emerald 高亮、加粗、大字号、badge 背景等方式强化视觉层级。

## 2. 设计目标

| 目标 | 验收标准 |
|---|---|
| 比赛详情页醒目 | AI 预测区标题的首选/次选比分使用带背景色的高亮 badge |
| 首页焦点战醒目 | 焦点战卡片「首选比分」数字使用 amber-300 + text-lg，次选 emerald-300 |
| 赛后复盘醒目 | 复盘卡片「预测比分」数字使用 amber-300，次选 emerald-300 |
| 色彩一致性 | 首选使用 amber 系（与卡片边框、标题呼应），次选使用 emerald 系（区分但不冲突） |
| 缓存刷新 | Service Worker 缓存版本升至 `wc2026-v7`，静态资源 query string 升级到 `?v=0.16.3` |

## 3. 修改文件

| 文件 | 改动 |
|---|---|
| `app/main.py` | `APP_VERSION` 常量升至 `0.16.3` |
| `app/static/js/app.js` | 3 处推荐比分展示改为 amber/emerald 高亮样式 |
| `app/static/index.html` | CSS/JS 静态资源 query string 改为 `?v=0.16.3`，Tab 注释更新 |
| `app/static/sw.js` | 缓存版本升至 `wc2026-v7`，注释更新为 v0.16.3 |
| `README.md` | banner、文档版本、API 速查、阶段描述、§10 变更日志同步到 v0.16.3 |

## 4. 关键设计决策

### 4.1 Amber + Emerald 色彩语义

- **首选比分**：`amber-300` + `bg-amber-500/20` + `border-amber-500/30` —— 与 AI 预测卡片已有的 amber 边框/标题形成统一视觉焦点。
- **次选比分**：`emerald-300` + `bg-emerald-500/15` + `border-emerald-500/20` —— 作为次要高亮，与首选形成冷暖对比但不过度抢眼。

### 4.2 升级 Service Worker 大版本

由于 `app.js` 中展示逻辑有实质改动，且需要确保用户浏览器加载最新 JS，因此将 `CACHE_VERSION` 从 `wc2026-v6` 升到 `wc2026-v7`，同时升级 `index.html` 的 `?v=0.16.3` 双重保险。

### 4.3 保持文本可读性

所有高亮文字均使用 `font-extrabold` 或 `font-bold`，并在深色 badge 背景上使用浅色文字，确保对比度。
