# v0.7.8 Calibration — Platt scaling 修主队高估

> 目的: 用 Platt scaling 对 G2/Elo/Blend 三种模型的 (ph, pd, pa) 做后验校准,
> 在 913 + 313 = **1226 场 walk-forward** 数据上验证是否能拉平 brier 并提高 accuracy。
> 关键动机: v3_g2 平均 ph=0.5105, 实际 home=0.493 → **高估 1.7 pp**
> 文件: 1 个服务 + 1 个端点 + 1 个测试文件 + 1 个 e2e

---

## 假设 / 边界

### 我假设
1. 1226 场是指 Hicruben 913 + StatsBomb 313 walk-forward 配对数据;不需要重新跑 G2 训练(已有 v0.7.6 补的 4 场)
2. Platt scaling 是 logistic regression 拟合 (z = a + b*logit(p)),输出仍是 (ph, pd, pa) 三元组,需要保留归一化(sum=1)
3. 校准模型按 outcome 维度 (home/draw/away) 独立拟合,即 3 个 (a, b) 对
4. 用 80% train / 20% time-ordered test split,避免 look-ahead bias
5. 校准后的预测只用于端点 `/api/elo/calibrated-predict`;旧端点(`/api/elo/predict-blend`)保留不动(v0.7.0a 用户兼容)
6. 不重写 prediction_log,不替换 v0.7.4 weight sweep 结论

### 边界 (不做的)
- ❌ 不重训 Glicko-2 (RD 调整不在 v0.7.8 范围)
- ❌ 不改 HOME_BONUS=70 (那是 v0.1.6 4 年回测最优值)
- ❌ 不引入 isotonic regression / beta calibration (Platt 是已知好 baseline)
- ❌ 不做 per-team 校准 (样本量不够,913 场 / 48 队 ≈ 19 场/队)
- ❌ 不动 Cockpit / 准确率 dashboard (留 v0.7.9 整合)

  - 端点支持 3 种 model 参数

## 验证目标 (验收门槛)

| 指标 | v3_g2 raw | v0.7.8 calibrated | 提升 |
|---|---|---|---|
| **brier (test 20%)** | 0.5120 | < 0.4950 | > 0.017 |
| **accuracy (test 20%)** | 0.6265 | > 0.6350 | > 0.85pp |
| **ph_mean 偏差** | +0.0175 | < +0.005 | < 0.013 |
| **ECE (期望校准误差)** | ~0.06 | < 0.04 | > 0.02 |

如果 calibrated 跑不出 brier 改进 ≥0.01,**标 v0.7.8-experiment 标签,不主推**。

## 风险 / Push Back

- 风险 1: 80/20 split 后 test 仅 ~245 场,accuracy 波动 ±3pp,可能 statistically noisy
- 风险 2: Platt scaling 对 calibration curve 单调性敏感,如果模型严重 mis-calibrated,brier 不降反升
- 风险 3: StatsBomb 313 场 walkforward 训练数据生成可能要写一次性脚本(若 elo_g2 数据缺)
- Push back: 主人若想"先 913 跑出 brier 改进再决定是否扩 1226",**做 913-only 路径**省 30 min,但 release 标题会改为 "v0.7.8a" 表明范围

主人默认 1226 路径,若 913 已经 30 min 内无 brier 改进我立即 push back。
