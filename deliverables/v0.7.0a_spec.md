# v0.7.0a "ModelBlend" Spec

> 状态:✅ Spec 已批准(主人选项 1,2026-06-16)
> 工时预估:2-3 小时(实际比预期少)
> 风险:低 — Elo 和 Glicko-2 接口都已就位,只是写"加权求和"

## 1. 目标

把 v1 (Elo) + v3 (Glicko-2) 两套预测**等权平均**为第三套"blend"预测,降低单模型方差。

**不做的:**
- ❌ 市场赔率融合(那是 v0.7.3)
- ❌ 训练 β 权重(无市场数据无法训练,等权 0.5/0.5 占位)
- ❌ Monte Carlo(那是 v0.7.1)
- ❌ 前端 UI(单独 v0.7.0a 之后,先用 raw JSON 验证)

## 2. API Contract

### 新增端点

```
GET /api/elo/predict-blend/{home_code}/{away_code}
  ?match_id=<int>           # 可选,自动写 prediction_log
  ?w_elo=<float 0-1>        # 可选,默认 0.5(Elo 权重)
  ?w_glicko2=<float 0-1>    # 可选,默认 0.5(Glicko-2 权重)
```

### 响应 Schema

```json
{
  "home": {"fifa_code": "BRA", "elo": 1842},
  "away": {"fifa_code": "ARG", "elo": 1810},
  "elo": {
    "model": "elo_dixon_coles_v1",
    "probabilities": {"home_win": 0.45, "draw": 0.27, "away_win": 0.28}
  },
  "glicko2": {
    "model": "glicko2_v1",
    "probabilities": {"home_win": 0.48, "draw": 0.25, "away_win": 0.27},
    "uncertainty": 0.12
  },
  "blended": {
    "model": "blend_elo_glicko2_v1",
    "weights": {"elo": 0.5, "glicko2": 0.5},
    "probabilities": {"home_win": 0.465, "draw": 0.26, "away_win": 0.275}
  },
  "predicted_outcome": "H",
  "data_source": "hicruben/world-cup-2026-prediction-model (913 matches walk-forward)",
  "data_as_of": "2026-06-11T...",
  "model_version": "v7a_blend"
}
```

### 错误响应

- 401/404:球队不在 Elo 或 Glicko-2 数据中(任何一个找不到,blend 不可用)
- 422:weights 之和不为 1(可加可不加,暂只校验 0-1 区间)

## 3. 算法

```python
blended_prob = w_elo * elo_prob + w_glicko2 * glicko2_prob
predicted_outcome = argmax(blended_prob)  # 'H' / 'D' / 'A'
```

**默认等权 0.5/0.5**(无市场数据无训练,简单透明)。
