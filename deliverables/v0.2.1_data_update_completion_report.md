## 教训（必须记录）

1. **不要重复说"修代码 1 行"**——可能该修复**早就在 v0.2.0 commit 里**了，重复说会误导主人认为有新 commit
2. **"调度任务跑了吗"必须看 scheduler 状态** + **"数据是否最新"必须看 last_updated_at**——不要凭印象
3. **DB 脏数据修复** ≠ **代码修复**——前者是 SQL 一次性 UPDATE，后者是 sync 写入路径
4. **修代码 vs 修 DB 现状要分清**——修代码只影响后续 sync，修 DB 现状需要一次性 SQL
5. **读盘 → 验证 → 修复 → 验证**——4 步不能省
