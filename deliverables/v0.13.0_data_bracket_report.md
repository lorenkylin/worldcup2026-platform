### 2. 数据去重与数据源统一

- 新增 `scripts/cleanup_v013_data.py`：
  - 默认 `--dry-run`，加 `--apply` 执行。
  - 删除占位球队（`Team A1` / `fifa_code=A1` 等）。
  - 合并重复球场（按规范化名称 + 城市 + 国家）。
  - 删除 `match_number` 不在 1-104 的孤儿比赛。
  - 重建 `standings`（只保留真实 48 队）。
- 在本地 DB 执行 `--apply` 后：
  - `teams` 96 → 48
  - `stadiums` 17 → 16
  - `matches` 105 → 104
  - `standings` 48 → 48

### 3. 模型与同步加固

- `app/models.py`：`Stadium.name_en` 增加 `unique=True`。
- 新增 Alembic 迁移 `alembic/versions/6faeb80b20c2_add_unique_constraint_on_stadiums_name_en.py`。
- `app/services/worldcup26_sync.py`：
  - `sync_stadiums` 增加 `(city, country)` + 规范化名称兜底匹配，避免 seed/sync 产生同名变体重复。
  - `full_sync` 末尾调用 `_cleanup_stale_rows`，自动清理占位球队和越界孤儿比赛。
