## 3. 关键设计决策

### 3.1 Dockerfile
- **基镜像 `python:3.13-slim-bookworm`**: Debian 12 基础, 体积 ~150MB
- **`WORKDIR /app`**: 标准应用目录
- **先复制 requirements 再复制代码**: 利用 Docker layer cache, 代码改不影响 pip 层
- **USER appuser (uid 1000)**: 安全最佳实践, 非 root
- **HEALTHCHECK**: Docker daemon 也会用, `curl -f /health`
- **CMD `uvicorn --workers 1`**: scheduler 单实例, 多 worker 会重复同步

### 3.2 docker-compose.yml
- **持久卷 `./data:/app/data`**: SQLite + sync_status.json
- **持久卷 `./logs:/app/logs`**: 应用日志
- **`restart: unless-stopped`**: VPS 重启后自动起
- **环境变量 `${ADMIN_TOKEN:-changeme-please-override}`**: 主人 .env 覆盖
- **json-file 日志 10m × 3**: 防止 VPS 磁盘爆

### 3.3 deploy.sh
- **`set -euo pipefail`**: 严格模式, 错就退
- **前置检查**: docker / docker-compose / .env
- **git pull + 显示 commit/tag**: 主人能看到部署的是哪个版本
- **`docker compose down` 停旧 + `build --no-cache` 重 build**: 避免缓存坑
- **60s 健康检查循环**: `curl -f /health`
- **失败提示**: 打印 `docker compose logs --tail=50` 排查命令

### 3.4 GitHub Actions CI
- **触发**: push master/PR/手动
- **跑 pytest (非 E2E)**: CI 无 live server, 跑 e2e 会假失败
- **alembic --sql 干跑**: 验证迁移可解析, 不实际执行
- **bash 语法 + import sanity**: 快速 fail-fast

## 7. 教训 (写入 MEMORY.md)

1. **bash -n 在 Windows 沙箱报错**: WSL 弹中文提示干扰测试, 改用 shlex + 内容检查
2. **Docker layer cache**: requirements.txt 先于代码复制, 改代码不重装 pip
3. **scheduler 单实例**: --workers 1 强制, 多 worker 会重复同步写库
4. **非 root 安全**: useradd + USER, 即使主人用 root 跑也能限制容器内权限
5. **alembic --sql 干跑**: CI 验证迁移可解析不实际执行, 避免 CI 库污染
