## 1. 目标

**问题**：match detail 页"⚔️ 历史交锋"卡片**只显示胜负条**（1胜1平0负），
看不到具体场次。用户想看两队**完整历史交锋**（2018+2022 世界杯 + 2026 完赛），
目前要靠记忆推断"近 N 次"是哪些场。

**目标**：建一个独立 H2H 详情页，展示两队**所有**直接对决的完整场次 + 胜负条，
作为可点击的"📜 完整历史"入口。

---

## 2. 范围与边界

### In Scope（做了的）
- 后端：新端点 `GET /api/h2h/{code1}/{code2}` + `GET /api/teams/{code}/h2h-opponents`
- 前端：详情页 `#/h2h/{code1}/{code2}` 路由 + `renderH2HDetail()` 函数
- 入口：match detail 页 H2H 卡片右上角加"📜 完整历史 →"链接
- 视角归一：code1 视角展示，code1/code2 互换时胜负方向自动翻转

### Out of Scope（没做的）
- 球队详情页 H2H 对手列表入口（`/api/teams/{code}/h2h-opponents` 端点已建好但前端没接，留 #P2）
- 3 队以上循环赛视图（13 队无数据，scope 大）
- 友谊赛 / 欧洲杯等历史数据（仅 2018+2022 世界杯 + 2026）
- H2H 比赛事件（进球 / 红黄牌 / 换人），H2H 表无此字段

### Assumptions（提前说清）
1. 路由 = `#/h2h/{code1}/{code2}`，展示**两队直接对决**而非单队历史
2. 数据源：H2HHistoricalMatch 表（111 场 2018+2022 种子）+ 2026 已完赛
3. 范围：1 个新 router（`h2h.py`，不复用 elo.py 因为业务不同）+ 1 个新页面 + 1 个入口
4. 不动 M1 / M2 / Cockpit / CSV 导出 / Bracket

## 3. 后端实现

### 3.1 新文件：`app/routers/h2h.py`（240 行）

**两个端点**：

```python
GET /api/h2h/{code1}/{code2}
  → 返回 {
      code1, code2,
      code1_team, code2_team,           # 球队信息（name_zh/flag_emoji/group）
      summary: {code1_wins, code2_wins, draws, total,
                current_2026, history},
      matches: [                        # 完整场次（按日期倒序）
        {match_date, code1_score, code2_score,
         code1_won, code2_won, draw, is_code1_home,
         competition, stage, source},
        ...
      ]
    }

GET /api/teams/{code}/h2h-opponents
  → 返回 {fifa_code, opponents_count, opponents: [...]}
  → 未来给球队详情页用
```

**关键技术**：
- **`_normalize_match()` 视角归一函数** — 任意主客方向的比赛归一为 code1 视角，
  code1/code2 互换时自动翻转胜负方向
- **数据源双查**：先查 2026 完赛（status=finished），再回退 2018/2022 种子
- **boundary 处理**：code1==code2 → 400，无队 → 404

### 3.3 修复两个 bug

| Bug | 原因 | 修复 |
|---|---|---|
| `AttributeError: 'Match' object has no attribute 'match_date'` | Match 表用 `kickoff_at` 而非 `match_date` | 改用 `m.kickoff_at` |
| `AttributeError: 'Match' object has no attribute 'competition'` | Match 表无 `competition` 字段（硬编码） | 写死 `"2026 FIFA World Cup"` + `stage` fallback 到 `Group {group_name}` |

## 6. 已知局限

1. **数据基础有限**：H2H 表仅 2018+2022 两届世界杯 + 2026 完赛。
   未含友谊赛 / 欧洲杯 / 美洲杯 / 世预赛等历史数据。
2. **本端"h2h-opponents"端点已建好但前端未接** — 留给 #P2 球队详情页扩展
3. **视角归一只在 API 层做** — 数据库里 2018+2022 比赛的主客是"原始主客"
   （多数为中立场），code1 视角归一不影响实际数据但会影响胜负条比例
