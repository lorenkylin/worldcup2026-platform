# v0.12.1 Spec — CI YAML 稳定性修复 + 真解析测试

> 2026-06-17
> **目标：堵住 v0.12.0 CI 配置中因 YAML 中文冒号导致的解析失败，并建立“真解析”回归测试。**

## 1. 背景

v0.12.0 新增 `.github/workflows/ci.yml`，其中 line 47 出现未引号的中文描述：

```yaml
# 错误示例（已修复）
run: alembic upgrade head --sql  # 干跑: --sql
```

`干跑:` 中的 `:` 被 PyYAML 解析为 key-value 分隔符，导致 `yaml.safe_load` 抛异常。该 bug 在 v0.12.0 的 grep 式测试中被漏掉（只检查字段存在性，未真正解析 YAML）。

## 2. 设计目标

| 目标 | 验收标准 |
|---|---|
| CI YAML 可解析 | `yaml.safe_load(ci.yml)` 不抛异常 |
| Compose YAML 可解析 | `yaml.safe_load(docker-compose.yml)` 不抛异常 |
| 回归测试 | 新增 2 个测试，真解析两个 YAML 文件 |
| 注释安全 | 含中文冒号的注释改用全角 `：` 或加引号 |

## 3. 修复内容

### 3.1 `.github/workflows/ci.yml`

- 将 `干跑:` 改为 `干跑：`（全角冒号），消除 YAML key-value 歧义。
- 统一所有中文注释使用全角冒号，避免未来再踩坑。

### 3.2 `tests/test_v012_deployment.py`

新增两个测试：

- `test_ci_workflow_yaml_parses`：真解析 `ci.yml`，断言顶层为 dict、有 `jobs` / `test`、触发器含 `push` / `pull_request`。
- `test_compose_yaml_parses`：真解析 `docker-compose.yml`，断言有 `services` 且至少 1 个服务。

## 4. 测试覆盖

- `tests/test_v012_deployment.py` 从 32 个断言扩展到 34 个，新增 2 个真解析测试。

## 5. 影响范围

- 仅基础设施测试与 CI 配置，无业务代码变更。
- 所有现有 pytest 不受影响。
