### 关键设计

#### 1. 性能预算(必达)

| 维度 | 目标 |
|---|---|
| 模拟次数 | 10000(可配 1000-50000) |
| 总比赛数 / sim | 72(组) + 16(R32) + 8(R16) + 4(QF) + 2(SF) + 1(3rd) + 1(F) = **104** |
| 总预测调用 | 10000 × 104 = **1.04M** |
| 单次预测延迟 | **< 0.01ms** (本机 LRU 内存命中) |
| 端到端目标 | **8-12s** |
| 内存峰值 | < 50MB |

**优化手段**:
- 服务启动时一次性把 Elo/G2 ratings **全量载入** `team_elo_cache[code] = int` 字典
- 比赛循环**不调** `predict_match_blend()` 包装层(里面 5+ 字典键访问),直接调 `match_prob()` + `predict_outcome()`,内联 blend 算术
- `random.choices()` 一次抽样 3 个候选(主/平/客),免 `random.random()` × 2
- 组赛模拟复用 v0.3.0 的 `_elo_to_lambda` + Poisson 思路(快)
- **不用 numpy**(没装且 overhead 反而大)

#### 2. 公平性保证

- **同一比赛**的"主队"由 FIFA 2026 bracket 指定(Match 73-88 的 home_team_id),沿用到 R16/QF/SF/F
- HOME_BONUS 沿用 `elo.HOME_BONUS`(50 Elo) — 中立场地调整为 0 的方案是 v0.7.1.1 微优化
- **加时/点球**简化:用 90 分钟**胜率 + 平率分配 35/65 给两队**(国际比赛加时进球率 ~0.3 球,合理近似)
- **3rd place playoff**:用 `predict_match_blend()` 单独算,与主 bracket 解耦

#### 3. 数据源

- 组赛复用 DB 真实 standings(已踢完的比分固定,不再重打),只模拟 `Match.status != "finished"` 的比赛
- 淘汰赛 73-104 的 home_team_id/away_team_id 来源:
  - 已写库的真实值(若有)
  - 否则从 R32_MATCHUPS 解析 placeholder
- Elo 用 hicruben(主)+ 缺数据 fallback statsbomb(2 队缺失见 v0.4.0)
- G2 用 hicruben 913 场 walk-forward 模型

    r32_slots = resolve_r32_matchups(...)  # 用 bracket_logic

#### 6. 加时 + 点球处理

```python
def knockout_winner(home_id, away_id, blend_probs) -> int:
    """淘汰赛 90 分钟 + 加时 + 点球."""
    h, d, a = blend_probs["home_win"], blend_probs["draw"], blend_probs["away_win"]
    # 90 分钟随机 → 平局进加时
    r = random.random()
    if r < h:
        return home_id
    if r < h + d:
        # 加时:重抽一次(主队 0.3 球,客队 0.2 球)→ 再平就点球 50/50
        # 简化:加时+点球合并为各 50% 概率
        return home_id if random.random() < 0.5 else away_id
    return away_id
```

## ❓ 风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| 10000 sims 跑超时 (>30s) | 中 | 用户等待久 | 预读 cache + 内联 blend,实测 < 15s |
| 某些队 G2 数据缺失 | 低 | 该队 fallback Elo,blend 变单模型 | 优雅降级(在响应里标记 `data_status`) |
| R32 解析失败(组赛未结束) | 中 | 端点 503 | 返回 503 + `detail="请等待小组赛结束"` |
| random 抽样偏差 | 极低 | 分布略偏 | 默认 10000 sims,误差 < 0.5% |
