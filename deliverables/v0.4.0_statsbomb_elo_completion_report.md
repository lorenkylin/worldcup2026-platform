## 一、目标与决策

### 1.1 原始目标
为 2026 FIFA World Cup 赛事分析平台引入 StatsBomb Open Data，训练一套新的 Elo 评级模型，作为现有 Hicruben 模型的补充或替代。

### 1.2 关键发现（可行性评估阶段）

| 维度 | 结果 |
|---|---|
| 可获取比赛数 | ~314 场（世界杯 2018/2022、欧洲杯 2020/2024、美洲杯 2024、非洲杯 2023）|
| 与 Hicruben 对比 | 仅 Hicruben 913 场的 34% |
| 时间覆盖 | 2018-2024，**缺少 2023-2026 友谊赛/预选赛/国家联赛** |
| 2026 参赛队覆盖 | 40/48（8 队无数据：IRQ、UZB、JOR、NZL、BIH、CUW、HAI、NOR）|
| 许可证 | 允许公开使用，**需标注 StatsBomb 并使用官方 logo** |

### 1.3 决策
**不替代 Hicruben**，将 StatsBomb 作为**可切换对比数据源**（方案 A）：
- Hicruben 继续作为默认主模型（覆盖 48 队、913 场、含近期状态）。
- StatsBomb 作为 `/api/elo/*?source=statsbomb` 的对比源，缺失球队自动 fallback 到 Hicruben。
- 新增 `/api/elo/compare/{home}/{away}` 并排展示两种模型预测。

## 三、技术细节

### 3.1 StatsBomb Elo 训练参数

```python
K_FACTOR = 60
INIT_RATING = 1500
HOME_BONUS = 0   # 大赛多为中性场地
DIXON_COLES_RHO = -0.13
BASE_LAMBDA = 1.35
```

### 3.2 球队名映射示例

```python
SB_NAME_TO_FIFA = {
    "Cape Verde Islands": "CPV",
    "Congo DR": "COD",
    "Iran": "IRN",
    "Korea Republic": "KOR",
    "United States": "USA",
    ...
}
```

### 3.3 Fallback 机制

StatsBomb 缺失球队（如 IRQ、NZL）调用 `get_statsbomb_team_elo()` 时返回 `None`，服务层自动降级到 Hicruben，并在 API 响应中显式标记：

```json
{
  "home": {
    "fifa_code": "IRQ",
    "elo": 1454,
    "rating_source": "hicruben_fallback",
    "fallback_reason": "StatsBomb missing IRQ"
  }
}
```

### 3.4 API 对比示例

`GET /api/elo/compare/ESP/BRA`：

```json
{
  "home_code": "ESP",
  "away_code": "BRA",
  "hicruben": {
    "probabilities": {"home_win": 0.52, "draw": 0.25, "away_win": 0.23},
    "expected_goals": {"home": 1.62, "away": 1.32}
  },
  "statsbomb": {
    "probabilities": {"home_win": 0.49, "draw": 0.26, "away_win": 0.25},
    "expected_goals": {"home": 1.45, "away": 1.38}
  }
}
```

## 六、已知限制

1. **StatsBomb 数据覆盖不完整**：缺少 8 支 2026 参赛队，依赖 Hicruben fallback。
2. ** compact 提取版本非官方全量**：沙箱无法访问 `raw.githubusercontent.com`，当前使用 WebFetch 提取的 309 场 compact 数据；在线环境可运行 `scripts/download_statsbomb.py` 获取完整 JSON。
3. **无独立回测**：StatsBomb 数据量不足以做 walk-forward 回测，未提供 statsbomb 版 backtest 端点。
4. **未接入 prediction 端点**：`/api/matches/{id}/prediction` 仍使用 Hicruben，避免影响比赛级预测稳定性。

---

## 七、后续建议

1. **v0.5+ xG 集成**：基于 StatsBomb event 数据做射门质量建模，而非仅做 Elo。
2. **多模型集成**：探索 Hicruben + StatsBomb 加权融合，提升预测稳定性。
3. **在线构建常态化**：网络环境恢复后，用 `scripts/download_statsbomb.py` 定期刷新 StatsBomb Elo。
