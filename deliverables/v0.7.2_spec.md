## 4. 同步服务: `app/services/odds_sync.py`

```python
def sync_odds_from_api(
    db: Session,
    client: Optional[OddsAPIClient] = None,
) -> dict:
    """拉取 + 解析 + 匹配 + 写库 4 步走.

    Returns:
        {
            "fetched": 12,        # The Odds API 返回 12 条
            "matched": 10,        # 匹配到 DB match_id 10 条
            "written": 30,        # 写入 match_odds 30 条 (10 场 × 3 公司)
            "unmatched": 2,       # 未能匹配的比赛
            "source": "api" | "mock",
        }

    匹配策略:
        1. 归一化队名 (lowercase + remove diacritics)
        2. 按 commence_time (UTC) 查最近 24h 内 kickoff_at 的 match
        3. home_team/away_team 与 Match.home_team.name_en 双向匹配

    写库策略:
        同 (match_id, bookmaker) 已有 → UPDATE home_win/draw/away_win + fetched_at
        不存在 → INSERT
        (因为 schema 没 UNIQUE 约束,用 `match_id+bookmaker` SELECT FOR UPDATE 锁)
    """
```

## 9. 风险与兜底

| 风险 | 兜底 |
|---|---|
| The Odds API 海外网络 | **Mock 优先**, 主人填 token 切真源 |
| 队名匹配失败(英文名差异) | `_normalize_team_name` 双向匹配 + 手动映射表 |
| rate limit 触发(每月 500 req) | 6h 调度 × 30 天 = 120 req, 完全够 |
| 比赛延期/取消 | sync_odds 写库时若 match.status != 'scheduled' 跳过 |
| 已有手工赔率被覆盖 | source != 'api' 的保留, source == 'api' 的 upsert |
