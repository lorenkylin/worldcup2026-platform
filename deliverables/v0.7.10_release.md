# v0.7.10 Release — Cockpit mini-card: G2 校准 brier 速览

## 核心改动

### 1. 轻量端点 `/api/elo/calibration-summary`
- **6 字段返回**: `training_samples` / `platt_full_fit_pp` / `platt_walkforward_80_20_pp` / `isotonic_walkforward_80_20_pp` / `cache_ttl_seconds` / `computed_at`
- **进程内 cache**: 6h TTL, double-checked locking, 第二次调用 <10ms
- **不依赖 home/away**: 全局摘要, Cockpit 频繁刷新零成本

### 2. Cockpit mini-card (Cockpit 路由 `/cockpit`)
- 标题 "🎯 G2 校准 brier 速览 (v0.7.10)"
- 3 列 card (Platt Full Fit / Platt 80/20 / Isotonic 80/20)
- **绿色表示 brier 改进 (pp ≥ 0)**, 红色表示 brier 倒退 (pp < 0)
- 数字带正负号 (+0.69 pp) 主人秒懂方向
- 底栏 "训练样本 913 场 · 缓存 6h"

### 3. 服务层 cache 函数
- `app/services/calibration.py` 新增 `get_calibration_summary()`
- 6h TTL, thread-safe (double-checked locking)
- 避免 Cockpit 频繁刷新时重复 913 场 walkforward (~100ms/次)

## 已知边界

1. **缓存失效率**: 6h TTL, 进程重启会清空 → lifespan startup 不会预热, 首次访问会重算
2. **mini-card 只读**: 不写 prediction_log, 不暴露单场 calibrated 概率
3. **数字符号约定**: +0.69 pp = brier 变好 (改进 0.69pp), -0.5 pp = brier 变差
4. **路由位置**: mini-card 在 `/cockpit` 路由 (主入口), 不在 `/` 路由 (home 简化版)
